요지 — 운영자 지시 스펙 그대로 영상 편집기를 상단 콘텐츠 제작 드롭다운에 신설했다. 한 도구(모달) 안에 탭 3개: 1. 영상 편집2. 자막 삽입3. 영상 프롬프팅
사전 조사 결과 이 레포에 기존 영상 편집 기능은 존재한 적이 없음(main·전 브랜치 70여 개·git 전 이력·과거 백업 문서 전수 검색) — 과거에 만졌다는 기억은 다른 도구/프로젝트의 것으로 판단, 합칠 레거시 없이 스펙 기준 신설이 정확한 작업이었다. 각 탭의 산출은 작업 사양서/프롬프트 텍스트(복사·전달용)이고, 실제 렌더링·AI 처리는 외부 도구 몫이다(앱 = 사양 조립 + 실시간 미리보기까지).

설계 요점

  • 산출 = 텍스트 사양서: 브라우저 단독으론 실제 인코딩·AI 처리가 불가하므로, 이 도구의 완성 산출은 "그대로 외부 도구(편집자·AI 서비스)에 전달 가능한 사양서/프롬프트"다. [생성] → 편집 가능한 텍스트 → [복사].
  • 기존 패턴 최대 재사용(§UI계승): 모달 셸·Esc·배경클릭 = 블로그 도우미 미러 · 칩/입력/노트 = nb-* · 탭 = .prog-tab · 스위치 = .sw · 버튼 = .btn 4변형 · 복사 = _nbCopyText · 임베드 검증 = _bpEmbedFromUrl(유튜브·Vimeo 화이트리스트).
  • 신규 프리미티브만 ve-*: 스테이지(16:9 프레임)·자막 오버레이·9분할 피커·컬러 스와치·레인지·컷 행 — 기존 등가물이 없는 6종만 신설, 색은 전부 토큰/기존 알파 변주(신규 hex 0).
  • 공용 PC 원칙: 상태 인메모리, 닫으면 소멸 + objectURL 회수(블로그 도우미와 동일). 단 열려 있는 동안 메뉴 재클릭은 리셋 없이 탭 전환(작업 유실 방지).
  • 재렌더 시 재생 지점 보존: 칩 클릭마다 전체 재렌더하되 비디오 currentTime·재생 상태 스냅샷 복원. 슬라이더(음량·자막 크기 등)는 재렌더 없이 오버레이/라벨만 직접 갱신.

한계 · 리스크

  • 임베드(유튜브·Vimeo) 소스는 미리보기 전용 — 음량 미리듣기·[현재] 위치 캡처는 파일/직링크에서만(iframe 제약, UI에 안내 문구).
  • 자막 미리보기는 근사 — 스테이지는 16:9 고정 프레임이라 세로 영상은 레터박스 기준 위치, 실제 번인 결과와 좌표가 다를 수 있음(사양서의 위치·여백 값이 정본).
  • 음량 100% 초과는 브라우저 미리듣기 한계로 100%까지만 들리고, 사양서에만 기록.
  • 모델 선택지는 260711 기준 통용 도구명 — 시장 변동 시 상수 배열(_VE_VMODELS 등)만 고치면 됨.
