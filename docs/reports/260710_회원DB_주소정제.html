1분해 규칙 — 시·군·구를 한 단위로

5v2 — 주소3 동화(洞化): 도로명 혼재 해소 (같은 날 후속)

도로명주소 행은 동 정보가 원천에 없어 주소3이 도로명이었다(22,693행). 공공 API 없이 DB 자체에서 매핑을 캐내 동·읍면 단위로 복원:

  • 씨앗 1 — 지번식 행(주소3=동)의 (우편번호→동): 신형 우편번호는 도로 구간 단위라 같은 우편번호 ≈ 같은 법정동.
  • 씨앗 2 — 도로명주소의 괄호 참고항목 (웅천동, 웅천지웰아파트)의 첫 요소 = 법정동 → ((시군구,도로명)→동).
  • 전파 — 우편번호로 동이 정해진 도로명을 사전에 증식. 모든 사전은 최빈값 2건 이상 + 85% 지배 시만 채택(오염 방지). 증거 우선순위 = 행 자신의 괄호 참고항목 > 자기 우편번호 > 도로명 최빈동(사전 최빈값이 구체 증거를 못 누름 — 검증단 교정 반영).

6앱 연동 — [회원 조회] 모달 (통합문서 → Worker → 앱)

기존 판매현황이 쓰는 GET /api/ops?sheet=… 경로가 「운영_」 프리픽스 시트를 임의로 읽는 구조라, 통합문서에 시트를 「운영_회원」 이름으로 복사하면 Worker 재배포 없이 앱에서 바로 조회된다.

  • 진입: 상단 「사업 실적」 호버 메뉴 ▸ 회원 조회 (관리자에게만 노출 — 개인정보라 admin 전용 게이트).
  • 기능: 이름·휴대폰(하이픈 무시)·주소 통합 검색 + 시도→시군구→동·읍면 3단 연동 필터 + 결과 표(상위 200행 표시). read-only — 회원 DB 원천은 홈페이지라 앱에서 수정 없음.
  • 로드: 최초 1회 전체 로드 후 메모리 캐시(재검색 즉시) · [새로고침] = 시트 재조회(fresh=1).
  • 검증: §분신술 5인(보안·회귀·데이터 정합·성능·UX) 적대적 감사 → 지적 전건 반영 후 만장일치 APPROVE — 핵심 반영: Worker 서버측 admin 게이트+no-store(클라 게이트 콘솔 우회 차단) · 7열×5,000행 청크 읽기(Graph 504·메모리 리스크 해소) · 로그아웃/유휴잠금 시 PII 소거 · zip 우선 동화 교정(193행) · 동별집계 오염 차단. 헤드리스 18/18 어서트 + check_design 통과(새 색 0).

⚠ 발효 조건: Worker(src/index.js)에 admin 게이트·청크 로더가 들어갔으므로 Cloudflare 재배포 후에 시트를 넣는 순서를 지켜야 안전(재배포 전에 시트부터 넣으면 그 사이 일반 로그인으로도 조회 가능한 창이 생김). 청크당 35k셀 = 기존 무사 시트와 동일 오더라 Graph 통과 확신도 높음(최종 확인은 실투입).
