1한 줄 원리 — "본다"가 두 종류야

"브라우저에서 대시보드를 본다"는 게 사실 서로 다른 두 가지 일이야. 이걸 안 나누면 "된다/안된다"가 세션마다 엇갈려.

🖨️

경로 A · "복사기"

원격 Claude(웹·코드 세션)가 앱을 새로 그려주는 것. 네 화면을 보는 게 아니라, 같은 앱을 자기 안에서 다시 띄움.

여기선 로그인 문(門)까지만 보여줄 수 있어.

로그인 前까지만
👀

경로 B · "어깨너머 친구"

"클로드 인 크롬"이 네가 이미 열어둔 진짜 탭을 같이 보는 것. 네가 로그인해 둔 그 화면 그대로.

로그인 後 실제 대시보드·실데이터를 봄.

진짜 화면 O
👉 그래서 결론 진짜 대시보드(숫자·데이터)를 Claude가 보게 하려면 경로 B야. 원격 코드 세션(경로 A)에 아무리 시켜도 네 탭은 못 봐 — 구조가 그래.

2경로 B 연결 순서 (핵심)

"클로드 인 크롬"이 대시보드 탭을 보게 만드는 순서. 위에서 아래로 한 번씩.

⚠ "메뉴엔 켜져 있는데 왜 원격은 못 써?" 클로드 인 크롬·데스크탑 커맨더는 네 PC에서 도는 로컬 커넥터야(하나는 네 브라우저를, 하나는 네 컴퓨터를 직접 만짐). 그래서 네 컴퓨터의 Claude(데스크탑 앱·브레이브)에서만 진짜로 작동해. 클라우드에서 도는 원격 코드 세션은 네 브라우저·PC에 손을 못 대서, 메뉴에 ON이어도 그 도구가 안 붙어(이 세션서 실측 — 목록에 아예 안 뜸). → 스샷 속 그 메뉴가 보이는 Claude(=네 로컬)에서 부탁해야 해.
  1. 브레이브에서 대시보드를 연다. 아래 주소로 들어가서 로그인까지 끝낸다(마이크로소프트 → PIN). 화면에 실제 대시보드가 떠 있어야 함.
    https://promo.yeulmaru.org/
  2. 그 탭에서 "클로드 인 크롬" 확장을 켠다. 브라우저 우측 위 확장 아이콘 → Claude → "이 사이트에서 사용 허용"(탭 접근 권한)을 준다. ※ 버튼 문구는 버전마다 조금 달라 — 핵심은 "이 탭 봐도 돼"를 허용하는 거야.
  3. 말 거는 그 Claude 대화에서 커넥터가 켜져 있는지 확인. 커넥터가 그 채팅에 ON 돼 있어야 도구가 뜬다(설치돼 있어도 채팅마다 꺼져 있을 수 있음).
  4. 같은 계정인지 확인. 브라우저 Claude와 확장이 같은 계정이어야 연결됨.
  5. 그 표면의 Claude에게 부탁한다. "이 탭(대시보드) 보고 ○○ 알려줘" 라고. 이제 진짜 화면을 같이 본다. ✅

연결 전 체크리스트 ✔

  • 브라우저에 Claude for Chrome 확장 설치·켜짐
  • 확장과 Claude가 같은 계정
  • 지금 이 대화(채팅)에서 커넥터 ON
  • 대시보드 탭 접근 권한 허용함
  • 그 탭이 이미 로그인돼 실제 대시보드가 떠 있음

3경로 A(원격 세션)는 뭐가 되고 뭐가 안 돼?

✅ 된다 앱 로그인 前 화면을 새로 그려서 스샷으로 보여주기. "앱이 브라우저에서 멀쩡히 뜨나?"는 확인 가능.

❌ 안 된다 로그인·실데이터. 원격 세션 브라우저는 바깥 인터넷에 못 닿아서, 마이크로소프트 로그인 라이브러리·데이터 서버(Worker)에 못 붙어. PIN을 줘도 로그인 자체가 불가.

⚠ 헷갈리기 쉬운 것 경로 A로 그린 화면 아래에 빨간 "MSAL 라이브러리 로드 실패" 토스트가 뜨는데, 이건 테스트가 바깥 CDN을 막아서 나오는 정상 산물이야. 실제 사이트의 버그가 아님 — 진짜 브레이브에선 안 떠.

라이브가 살아있는지만 궁금하면 브라우저 없이도 확인돼:

curl -sS -o /dev/null -w "%{http_code}\n" https://promo.yeulmaru.org/

→ 200 나오면 살아있는 거야.

4원리 — 왜 원격은 진짜 화면을 못 볼까?

🏢 비유: 원격 Claude는 인터넷이 막힌 사무실 안에 있어. 바깥으로 나가려면 정문 경비실(프록시) 하나만 통과할 수 있어.

경비실은 내 심부름꾼(curl)은 통과시켜 주는데, 브라우저가 직접 나가려니 문을 탁 닫아버려(연결이 리셋됨). 그래서 원격 브라우저는 마이크로소프트 로그인 창도, 데이터 서버도 못 불러와.

반대로 경로 B("클로드 인 크롬")는 네 컴퓨터의 브레이브 안에서 도니까, 바깥 인터넷도 네 로그인도 이미 다 돼 있는 상태 그대로를 봐. 그래서 되는 거야.

한 줄 정리: 원격 = 바깥망 막힘 → 로그인 前 그림만. / 네 브레이브 = 다 열려 있음 → 진짜 화면.

5"연결했는데 안 보여" — 이 순서로 점검

  1. 대화(채팅)에서 커넥터 ON인가? — 제일 흔한 원인. 설치돼 있어도 이 채팅엔 꺼져 있을 수 있음.
  2. 탭 권한 줬나? — 확장 아이콘 눌러 그 사이트 접근 허용.
  3. 탭이 로그인 상태인가? — 로그아웃/세션만료면 로그인 문만 보임. 다시 로그인(MS→PIN).
  4. 같은 계정인가? — 확장과 Claude 계정 불일치면 연결 안 됨.
  5. 원격 세션에 시킨 건 아닌가? — 메뉴에 ON이어도 원격(클라우드) 코드 세션엔 이 커넥터가 안 붙음(로컬 커넥터라서). 네 PC의 Claude(스샷 속 그 메뉴가 보이는 쪽)에서 해야 함.
