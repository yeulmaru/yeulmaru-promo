1. 뭘 만들었나 (한 줄)

공연 운영계획(안) PPT를 모달에 끌어다 놓으면 공연 개요·가격 정책·할인 정책을 자동으로 읽어 폼을 채우고, 부족한 항목(오픈 일시 등)만 직접 입력하면 NHN티켓링크 판매대행 의뢰서(xlsx)가 원본 서식 그대로 채워져 다운로드된다.

일정 관리 ▸ 🎫 티켓 등록 신청→ PPT 업로드 (자동 채움 12항목)→ 부족분 입력 (오픈일시 등)→ 의뢰서 엑셀 다운로드

데이터는 어디에도 저장 안 함(시트·서버 무관) — 브라우저 안에서만 변환. 같은 PPT 양식(운영계획안)이면 어떤 공연이든 동작.

3. 왜 이렇게 만들었나 (설계 결정)

  • zip 레벨 셀 패치 — 의뢰서엔 체크박스 폼 컨트롤 81개·메모 10개·병합 300여 개·수식이 있다. SheetJS/ExcelJS로 재생성하면 이게 다 날아가서, 템플릿 xlsx(zip)의 XML에서 대상 셀 값만 문자열 치환한다. 원본 120개 엔트리 무손실.
  • 시트2는 수식 캐시 갱신 — 공지문(시트2)의 제목·기간·시간·장소·협찬·기타정보는 시트1 참조 수식. 수식은 보존하고 캐시 값만 갈아끼우며, fullCalcOnLoad를 켜서 엑셀이 열 때 재계산하게 했다(미리보기 앱 대비 이중 안전).
  • 장르 = 드롭다운 sel + 체크박스 checked 패치 — 시트1 장르는 폼 콤보(선택 인덱스), 시트2 장르는 체크박스 8개. 둘 다 VML과 ctrlProp 양쪽을 동기 패치.
  • 템플릿은 공연별 셀만 공란화 — 업로드해준 조재혁 완성본에서 공연 정보만 비우고 회사 보일러플레이트·체크 상태는 보존. reference/ticketlink_template.xlsx로 커밋(gitignore 좁은 예외).
  • JSZip은 지연 로드 — 차트(plotly)와 같은 패턴. 티켓 기능을 안 쓰면 로드 안 됨(기존 화면 성능 영향 0).

5. 한계·운영 노트

  • PPT 양식 전제 — 「공연명/일시/장소/티켓 가격/관람 시간/관람 연령 :」 키-값 줄과 좌석·할인 표가 있는 운영계획(안) 포맷. 벗어나면 해당 항목만 빈칸으로 남고 직접 입력하면 된다(모달이 죽지 않음).
  • 좌석 등급 6행 고정 — 양식 자체가 6행. 넘치면 엑셀 4번 시트(등급 및 권종)에 수기.
  • 시트1 체크박스 51개는 회사 표준 고정 — 조재혁 실제 제출본의 상태를 계승(잔여석 노출·클린예매·문화비 소득공제 등). 공연마다 바뀌는 값이 생기면 엑셀에서 직접 토글.
  • 시트5(현장 지원) — 공연명·기간·장르만 자동, 나머지는 티켓링크와 협의 후 수기(원본도 공란 제출).
  • 개인정보 위생(해결) — 템플릿에서 개인 휴대폰 3건·이메일 2건·티켓링크 ID·저장경로 메타를 전수 제거(공유문자열·하이퍼링크 잔재 포함). 유지 = 이름(공문 수준)·사무실 국번·사업자번호(공부 정보)·주소. 매 생성 시 담당자 연락처가 필요하면 모달에 입력하거나 엑셀에서 채우는 구조 — 완성본과 100% 동일하게 나오게 하려면 이 값들을 어딘가 박아야 하는데, 공개 레포엔 부적합해서 이렇게 갔다. 다른 방식 원하면 말해줘.
