① 목록 뷰 — 판매중 공연 카드

픽스처 데이터 헤드리스 실측 화면. 상단: 판매중 3 · 어제 합계 ▲+210석 · 단순평균 점유율 · 기준일(최신 일일입력). 왼쪽 캘린더 맨 위 = 접힌 지난 주(6/30~7/5) 스트립.
  • 카드 구성(분석 보드 스크린1 계승): 공연명 · D-day(색상 임박도) · 장르·수익성 축·공연일 · 현재 점유율(큰 숫자) · 어제대비 ▲▼ %p · 최근 입력분 스파크라인 · 목표 틱(살몬) 진행 바 · 판매/오픈석 · 목표 갭(달성 초록/미달 레드)
  • 정렬 = 공연일 임박순(개막해 진행 중인 D+가 최상단), 동률이면 점유율 낮은 순(주의 대상 위로)
  • [분석] 버튼 = 분석 보드(개선판) · [새로고침] = 강제 재로드 · 일일입력 저장 시 자동 갱신

② 인라인 개별 분석 — 카드 클릭 (캘린더 안 가림)

시카고 카드 클릭 → 레일 안에서 개별 분석. 캘린더는 그대로 — 왼쪽에서 홍보계획 짜면서 오른쪽으로 판매 상황 판단.
  • 분석 보드 드릴과 동일 로직: KPI 4장(현재 누적 점유율/어제대비/목표·차이/공연일 D-day) · D+ 누적 점유율 미니 차트(목표 점선, 점 hover=일자·석수) · 상황(목표 대비+또래 중앙값) · 효과 채널 · 최대 레버 · ▶ 다음 액션
  • [← 목록] 복귀 · [크게 보기] = 분석 보드 모달 드릴(또래 비교 트레이·그날 왜 표까지)

③ 날짜 클릭 패널과의 공존

날짜를 클릭하면 상세 패널이 레일 자리를 정확히 덮는 폭으로 열림 — 캘린더는 3/4 그대로(안 밀림). 닫으면 레일 복귀. 1200px 이하 화면은 레일 숨김 + 기존 360px 동작 그대로.

데이터 흐름 · 동작 규칙

  • 데이터 = 기존 _salesBuild() 재사용(운영_일일입력·공연마스터·회차상세·운영대장) — 판매/분석 보드와 완전 동일 출처·수식(점유율·어제대비 임계 0.04%p·목표 50 폴백). 레일이 먼저 채우면 두 보드는 재fetch 없이 즉시 열림.
  • 「판매중」 = 공연마스터의 티켓오픈일 ≤ 오늘 ≤ 판매종료(+마지막 공연일 보정) — 기존 판정 그대로.
  • 실시간이 아니라 일일입력 최신분 기준 — 헤더에 「기준 M.D」 명시. 어제대비도 입력 시점 기준.
  • 부트 시 백그라운드 병렬 로드(캘린더 렌더 비차단) · 좁은 화면(≤1200px)에선 데이터도 안 당김(넓히면 자동 재시도).

검증 — 분신술 5인 + 재검증 2인 + 헤드리스 실측

한계 · 남긴 것 (정직 표기)

  • 전시 미포함 — 레일은 공연 판매추이만(전시 진행 현황은 분석 보드 전시 탭에 있음). 원하면 후속으로 전시 카드 추가 가능.
  • 일일입력이 끊긴 공연은 스파크·어제대비가 「수집 전」으로 정직 표기.
  • LOW 잔여(감사 기록): 일일입력 저장 시 보드+레일이 같은 시트를 이중 fetch(무해·네트워크만) · api 타임아웃은 미도입(세대 토큰으로 교착만 해소) · 마스터 없는 공연의 판매중 판정은 공연일 기준(기존 동작 상속).
