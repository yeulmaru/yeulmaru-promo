## 1. 지시·의도·판단 기록 (왜 이렇게 이해했나)

### 1-1. 지시의 분해
1) **이식**: 예울마루 프로모의 애니어그램을 노뮤트 에디터로 옮겨라(7분 컷).
2) **파악**: "뭘 반영해야 할지"를 스스로 알아내라 — 수용처의 규범·구조에 정식 편입되기 위한 갭 전수 도출 위임.
3) **계획 보고**: 추후 계획을 3만자 분량으로 보고하라.
4) **검증**: 그 계획이 "제일 효율적인지"를 다른 페이블5가 6분 안에 적대 검증하라.

### 1-2. 의도 해석
- 애니어그램 툴킷은 예울마루의 `playground/`(시안 구역)에 있었다. 노뮤트 에디터는 운영자의 **도구 허브**(뷰어+스튜디오들이 사는 라이브 웹앱)다. 본질 = **"실험 구역에서 잘 만들어진 도구를, 도구들이 정식으로 사는 집으로 옮겨 상시 접근 가능하게 하라"**는 자산 재배치.
- "뭘 반영해야 할지 알아내" = 편입에 필요한 전체 항목(캐시 계약·게이트 등재·진입 동선·계정 축 등)을 실측으로 갭 리스트화하라는 위임.
- **⚠v2 해석 명시(위원5 지적 수용)**: 지시의 "페이블 5인"·"다른 페이블5인"은 「페이블5(모델명)+인(人)」으로도 「5인(5명)」으로도 읽히는 중의문이다. 본 세션은 {작업 주체 = 메인 세션(레포 변경은 메인만 — 평의회 규약 계승) · 검증 주체 = **페이블5 모델 5인 병렬**}로 해석해 실행했다. 5인 채택 근거: [E10] 평의회 문화(N인 병렬 적대 검증)의 적용 + 숫자 5가 지시문에 반복 등장. 1인 해석이었대도 5인은 상위 호환(검증 강도↑)이며, 이 병렬은 지시문의 "검증" 명령이 직접 발주한 것이라 [8](임의 병렬 금지) 비위반.

### 1-3. 착수 판단
- 이식 방식: 원본 README가 답을 명시 — "이 폴더는 자기완결형이라 **통째로 어디로 옮겨도 그대로 빌드됩니다**(경로가 상대 기준)". → A안(통폴더) 채택, 상세 논증 6장.
- 수용처 위치: `viewer/` = Cloudflare Pages 배포 루트(증거: `_headers`·`manifest.json`·`favicon.ico`·`sw.js`가 viewer/에 존재) + 스튜디오들이 독립 HTML 표면으로 사는 문법. → `viewer/enneagram/`.
- 디자인: 노뮤트 [16] 팔레트 예외 조항("도구 스튜디오들의 차분한 툴톤은 의도 — index 색 재색칠 영구 금지")의 보호 대상으로 편입. **재색칠 0, 원본 1:1**(기틀 3조 2항). 위원5 실측: 이식본 diff = 원본과 완전 동일 — 창작 0 기계 확인.

- **서비스워커(sw.js) — ⚠v2 신규 실측 확정**: `SHELL_PATHS = ['/', '/index.html']` **화이트리스트** 방식 + "여기 없는 HTML은 SW가 손 안 댐" 주석 + `req.mode !== 'navigate'` 불간섭 — 즉 `/enneagram/` 진입을 SW가 가로채거나 캐시할 가능성 **0 확정**(메인 세션 grep + 위원1·3 독립 실측 = 3중 일치). v1이 이 축을 통째로 미검토한 것이 위원1·3 공통 지적이었고, 본 실측으로 종결.
- **PWA(manifest.json) — ⚠v2 신규 실측**: `start_url "."`·`id "/"` = 노뮤트 앱 아이덴티티 전용. 에니어그램은 별도 설치 대상이 아니므로 **등재 불요 확정**(위원1).
- **접근 게이트(`functions/_middleware.js`) — ⚠v2 신규 실측(위원3)**: pages.dev 전 요청 → **apps.nomute.kr 301** + **Cloudflare Access 인증**. 실효 진입 = `https://apps.nomute.kr/enneagram/`(인증 후). 콘텐츠 노출면 = 인증 사용자 한정. P0-1 체크포인트의 curl 대상도 이 최종 URL로 교정.
- **보안 헤더**: `X-Content-Type-Options`·`X-Frame-Options: SAMEORIGIN`·CSP **Report-Only**(`font-src 'self' data:` — base64 폰트 허용 확인). 교재 iframe은 srcdoc(동일 문서)이라 SAMEORIGIN 무관. CSP 강제화 시 재검증 대상은 R5 유지.

## 6. 대안 비교 — 왜 이 방식이 "제일 효율적"인가 (⚠v2 논거 교정판)

### 6-1. 후보 4안
**A. 통폴더 이식(채택)** — 15파일 무변형 복사, 신규 폴더 단일 커밋.
**B. iframe 원격 임베드** — 코드 반입 없이 노뮤트 화면에서 예울마루 URL을 iframe으로.
**C. 소스만 이식 + CI 빌드** — 소스만 커밋, all.html은 배포 시 빌드 생성.
**D. 노뮤트 네이티브 재작성** — index 토큰·부품 문법으로 재구현.

### 6-3. 논증 (⚠v2 — 위원2 반증 반영 전면 교정)
- **B안 탈락 근거 교정**: v1은 "임베드할 원격 URL 자체가 없다(playground 비노출)"를 결정적 근거로 썼으나 이는 **사실 오류**였다 — `promo.yeulmaru.org/playground/enneagram/enneagram_all.html` = HTTP 200 서빙 중(위원2 curl 실측 · 예울마루 [16] playground=서빙 경로 명시). **교정된 탈락 근거**: ① "노뮤트로 이식시키셈"의 문언 자체가 코드 반입을 요구 — 임베드는 지시 미충족 ② 크로스레포 가용성 결합(예울마루 배포 사고 = 노뮤트 도구 동반 다운) ③ 이중 관리가 해소 불가능한 구조로 고착(호스트=노뮤트, 콘텐츠=예울마루). 결론 불변, 근거 교체.
- **C안 후순위 근거 교정**: v1의 "Pages 빌드 커맨드 신설 필요" 역시 부정확 — 노뮤트에는 `build-viewer.mjs`가 **이미 Pages 빌드 명령으로 돌고 있다**(파일 헤더 실측 · 위원2). 정확한 쟁점: **기존 공유 빌드에 python 단계를 결합하면 에니어그램 빌드 건강이 전 표면 배포의 인질**이 된다 + 산출물 커밋이 현행 지시(.gitignore 주석 실측)라 C의 전제(산출물 비커밋)부터 현행과 상충. 단 A는 소스를 전부 동반 이식했으므로 **C로의 전환 경로는 열려 있다**(비가역 아님).
- **D안 탈락**: 최대 비용 + 규범 정면 위반. 논외.
- **A안 채택 골격(불변)**: ① 원작자 설계 의도("통째로 옮겨도 그대로 빌드") ② 수용처 관례(독립 표면 병렬) ③ 변형 0 = 게이트가 기계 증명 ④ 어떤 후속안으로도 확장 가능한 무손실 출발점. 위원2 총평: "7분 컷 단독으로 B·C·D가 전멸하는 과잉결정 구조 — 어떤 전제를 쳐도 채택은 뒤집히지 않음."
- **공식의 한계 자인(⚠v2 · 위원2)**: 효율 = (지시 충족도×확장성)/(시간×신규 리스크)에서 A만 실측값이고 B·C·D는 추정치 — 측정 비대칭이 있는 수사적 골격임을 명시한다. 유지비(반복 비용) 항은 공식 밖 — 이중 사본 비용은 D-2로 별도 처리. 제5안 탐색(위원2): 서브모듈·별도 Pages 프로젝트·심링크 전부 A 이하 판정, 유일 변형 "배포물 2파일만 이식"은 P1-1이 소스를 요구하므로 로드맵상 A가 지배.
