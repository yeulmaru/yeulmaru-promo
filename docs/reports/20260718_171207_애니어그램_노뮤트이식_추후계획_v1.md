## 1. 지시·의도·판단 기록 (왜 이렇게 이해했나)

### 1-1. 지시의 분해
지시는 네 요구를 담고 있다.
1) **이식**: 예울마루 프로모의 애니어그램을 노뮤트 에디터로 옮겨라(7분 컷).
2) **파악**: 옮기면서 "뭘 반영해야 할지"를 스스로 알아내라 — 즉 이식이 단순 복사로 끝나지 않고 수용처(노뮤트)의 규범·구조에 정식으로 편입되려면 무엇이 더 필요한지 전수 도출하라.
3) **계획 보고**: 추후 계획을 3만자 분량으로 보고하라.
4) **검증**: 그 계획이 "제일 효율적인지"를 다른 페이블5가 6분 안에 적대 검증하라.

### 1-2. 의도 해석 (디자인적 사고)
- 애니어그램 툴킷은 예울마루 프로모 레포의 `playground/`(시안·실험 구역)에 있었다. 플레이그라운드는 본질적으로 "임시 거처"다. 반면 노뮤트 에디터는 운영자의 **도구 허브**(뷰어 + 스튜디오 여러 개가 사는 라이브 웹앱)다. 즉 이 지시의 본질은 **"실험 구역에서 잘 만들어진 도구를, 도구들이 정식으로 사는 집으로 옮겨 상시 접근 가능하게 하라"**는 자산 재배치다.
- "뭘 반영해야 할지 알아내"는, 운영자 본인도 수용처 편입에 필요한 전체 항목(캐시 계약, 게이트 등재, 진입 동선, 계정 축 등)을 다 나열할 수 없으니 **실측으로 갭 리스트를 만들어 오라**는 위임이다.
- "제일 효율적인지 검증"은 [E10] 평의회 문화의 연장 — 계획의 자기만족을 차단하는 적대 검증 요구다. 이번 건은 운영자가 검증 인원 모델을 **페이블5**로, 시한을 **6분**으로 직접 지정했으므로 이 실지시가 [E10]의 기본값(OPUS 4.8 8인)보다 우선한다.

### 1-3. 착수 판단
- 이식 방식은 원본 README가 스스로 답을 주고 있었다: "이 폴더는 자기완결형이라 **통째로 어디로 옮겨도 그대로 빌드됩니다**(경로가 상대 기준)". 설계 의도 자체가 통폴더 이식이다. → A안 채택(상세 논증 6장).
- 수용처 위치는 노뮤트 CLAUDE.md [15]·[16] 실측으로 결정: `viewer/` = Cloudflare Pages 배포 루트(증거: `_headers`·`manifest.json`·`favicon.ico`·`apple-touch-icon.png`가 전부 viewer/에 있음), 스튜디오들이 각각 독립 HTML 표면으로 사는 문법. → `viewer/enneagram/`.
- 디자인 처리: 노뮤트 【레포 바인딩】 [4] 팔레트 예외 조항 — "도구 스튜디오들(index 외 뷰어)의 차분한 툴톤은 의도라 index 색으로 재색칠 = **영구 금지**". 에니어그램은 자체 완결 디자인(다크 네이비 `#1A1A2E` 축 + 포인트 `#4A4DE7`)을 가진 독립 톤 표면이므로, 이 조항의 보호 대상으로 편입하는 것이 정본 해석이다. 즉 **재색칠 0, 원본 디자인 1:1 계승**(기틀 3조 2항 "이미 있는 디자인을 따른다"). 임의 창작 0.

### 2-2. 앱 아키텍처 (기능을 만질 사람을 위한 지도)

### 3-3. 배포·보안 계약 실측 (`viewer/_headers`)
- 캐시 규칙: 자주 바뀌는 HTML·JSON은 경로별로 `Cache-Control: no-cache`(매 로드 재검증 — 새 배포가 하드새로고침 없이 반영). 스튜디오들(`/ly`·`/k`·`/edit`·`/song`·`/nb` 등)이 **개별 경로로 전부 등재**되어 있다. 260712에 신설 도구 뷰어(no-cache 누락분)를 소급 등재한 이력 주석도 남아 있다. → **`/enneagram/` 경로는 아직 미등재** = 형제 뷰어와 같은 계약을 받으려면 등재 필요(→ 5장 P0-1). 미등재 상태의 실효 리스크는 "재방문 시 구버전 캐시가 잠시 보일 수 있음" 수준(Cloudflare Pages 기본 캐시는 HTML에 보수적이라 치명적이지 않지만, 형제 계약과 다름 = [4] 통일 원칙 위반 상태).
- 보안 헤더: 전 응답에 `X-Content-Type-Options: nosniff` · `X-Frame-Options: SAMEORIGIN` · `Referrer-Policy` · **CSP는 Report-Only**로 `font-src 'self' data:` 포함. → 에니어그램의 base64 데이터 URI 폰트는 **현행 CSP 초안과도 충돌 없음**(실측 확인). 단 CSP가 훗날 강제 모드로 승격되면 `frame-src`·`srcdoc`·인라인 스크립트 의존이 있는 이 앱도 재검증 대상(→ 7장 R5).
- `X-Frame-Options: SAMEORIGIN` — 에니어그램의 교재 iframe은 `srcdoc`(동일 문서 내)이라 무관. 문제없음.

### 3-4. 디자인 규범과의 관계
- 노뮤트 [16] 바인딩: 디자인 진입점 = `docs/디자인기틀_SSOT.md` §0 → `viewer/index.html` :root 토큰 + CII 컴포넌트인덱스. **단, 팔레트 예외 조항이 도구 스튜디오의 독립 툴톤을 명시 보호**한다("index 색으로 재색칠 = 영구 금지").
- 에니어그램은 "index의 부품을 계승해 새로 만드는 화면"([4-1] 3단 문법의 1단)이 아니라, **완성된 외부 앱의 편입**이다. 유형상 발행 콘텐츠/독립 표면 축에 속하므로 index 토큰 강제 대상이 아니다. 이번 이식에서 색·마진·정렬 등 디자인 값 변경 0 — 3조 3항(임의 규칙 창작 금지)·[4](값 창작 금지) 위반 소지 0.
- 다만 **메인 뷰어에서 에니어그램으로 들어가는 진입 링크**(런처 버튼 등)를 새로 만들 경우, 그 버튼 자체는 index 화면의 신규 요소이므로 [4-1] 1단("새 화면 신설 = index `#genidlg` `.geni-row`/`.geni-opt` 계승") + [4] 3중 안전장치(단일 커밋 분리 + ⚠배치 보고 + 해석 확인 문구)를 따라야 한다(→ 5장 P0-3).

- 판단(권고): 게이트 축별로 갈라야 한다 — ① **경로 참조 게이트(check_refs)**: 자동 포함됨(이번 rc=0에 신규 폴더 포함 통과). 추가 등재 불요. ② **디자인 토큰 게이트**: 에니어그램은 팔레트 예외 축(독립 톤)이므로 index 토큰 baseline에 **넣지 않는 것이 정본** — 넣으면 오히려 원본 색이 '위반'으로 오탐된다. ③ **smoke(UI 기하 실측)**: 등재 가치 있음 — 단 "이 표면을 노뮤트 세션이 앞으로 수정할 때"만 의미가 있으므로, P1-1(계정 축) 작업에 착수하는 시점에 `shared/smoke_enneagram.js`(탭 7개 존재·클릭 라우팅·저장 라운드트립 어서션)를 '대기' 티어로 신설하는 것을 권고. 헤더에 담당 표면 선언(shared/ 파일 문법 계승).
- 왜: 등재 누락 = 사각(규정), 과잉 등재 = 오탐 유지비([4-1]이 명시한 비채택 사유). 축별 선긋기가 규정 취지에 맞는 최고 효율.
- 공수: smoke 신설 시 1시간. 체크포인트: smoke rc=0 + '대기' 티어 헤더 세칙 기재.

### P2 — 중기 (정본 관리 · 결정 필요)

**P2-1. 정본(SSOT) 지위 이관 선언 ⚠(운영자 확인 2건째)**
- 무엇: 현재 애니어그램이 **두 레포에 동시에 존재**한다(예울마루 playground = 원본, 노뮤트 viewer = 사본). 이 상태를 방치하면 언젠가 한쪽만 고쳐져 갈라진다(드리프트 — 이 레포 문화가 가장 싫어하는 상태).
- 권고안: **노뮤트를 정본으로 선언**하고, 예울마루 쪽은 ⓐ 폴더에 `README_이관.md` 1장("정본은 nomute-editor viewer/enneagram — 여기는 260718 동결 스냅샷, 수정 금지") 추가 후 동결 보존, 또는 ⓑ 폴더를 삭제하고 포인터 문서만 남김. **ⓐ를 권고**: 삭제는 비가역·파괴라 [8]상 사전 문답이 필요하고, 예울마루 playground는 원래 시안 보관 구역이라 동결 스냅샷 보존이 자연스럽다. 판단 근거: 도구의 생활권(유지보수·기능 추가·게이트·계정 축)이 전부 노뮤트에 있다.
- 실행: 운영자 답 1줄("노뮤트 정본, 예울마루 동결" 승인) → 예울마루에 이관 README 커밋 → 이후 모든 수정은 노뮤트에서만.
- 공수: 10분. 체크포인트: 두 레포 README에 정본 방향 명시 상호 일치.

**P2-2. `apps/enneagram/` 두뇌 문서 신설**
- 무엇: 노뮤트 관례상 각 도구는 `apps/<이름>/`에 운영 지침(두뇌)을 둔다(14종 실측). 에니어그램도 `apps/enneagram/README.md`(또는 지침.md)를 만들어 원본 README의 핵심(아키텍처·빌드법·기능추가 4단계·주의사항)을 승계하고, 노뮤트 고유 사항(계정 축 배선 상태·게이트 등재 현황·정본 선언)을 추가.
- 왜: viewer/enneagram/README.md는 원본 승계본(예울마루 시점 서술)이라, 노뮤트 시민으로서의 운영 정보(어느 게이트에 등재됐고, 데이터가 어디 사는지)는 apps/ 축이 정위치. 다음 세션 클로드의 탐색 비용을 없앤다.
- 공수: 30분. 체크포인트: apps/ 목록에 enneagram 표시 + 내용에 P0~P1 반영 현황표.

**P2-3. 콘텐츠 라이선스 유의 승계**
- 무엇: 강의 자막 50편은 원 유튜브 강의의 콘텐츠다. 원본 README의 "개인·팀 학습용, 외부 공개 시 원저작권 유의"를 노뮤트 쪽 문서(P2-2 두뇌 문서)에 그대로 승계 명기하고, 라이브 노출 범위(nomute-editor.pages.dev = 운영자 도구 허브, 불특정 다수 홍보 아님)를 기록.
- 왜: 접근 도메인이 바뀌었으니 유의사항의 소재지도 따라와야 한다. 법적 조치가 아니라 **인지 유지 장치**다.

**P2-4. 대용량 파일 위생 (관찰 항목 — 당장 조치 불요)**
- 무엇: 이번 이식으로 노뮤트에 ~8.6MB(그중 all.html 4.2MB·폰트 2.6MB·자막 1.4MB)가 추가됐다. 노뮤트는 이미 대용량 자산(스킬 파일 4~6MB급 다수, _versions 1,412개 디렉토리)이 있는 레포라 비중은 작다. 원본 README는 "산출물은 빌드로 생성(gitignore) + LFS 고려"를 권고했지만, 현행 지시 이력("파일 전체를 깃에")대로 전량 커밋 상태를 승계했다.

## 6. 대안 비교 — 왜 이 방식이 "제일 효율적"인가 (검증단 공격 대상)

착수 시점에 4개 안을 놓고 비교했다. 채택 = A안.

### 6-1. 후보 4안

**A안. 통폴더 이식 (채택)** — 원본 폴더 15파일을 `viewer/enneagram/`에 무변형 복사, 신규 폴더 단일 커밋.
**B안. iframe 원격 임베드** — 노뮤트에 코드 반입 없이, 노뮤트 화면에서 예울마루 쪽 URL을 iframe으로 띄움.
**C안. 소스만 이식 + CI 빌드** — 소스 4파일만 커밋하고 all.html은 gitignore, Pages 빌드 단계에서 python 빌드를 돌려 산출.
**D안. 노뮤트 네이티브 재작성** — 앱을 노뮤트 index 토큰·부품 문법으로 재구현해 스튜디오 탭으로 통합.

### 6-3. 논증 요지
- **B안 탈락**: "이식"의 사전적 요구(코드가 노뮤트로 옮겨 옴)를 충족하지 못한다. 결정적으로 예울마루 playground 경로는 라이브 서빙 대상이 아니어서 임베드할 원격 URL 자체가 없다 — 만들려면 예울마루 배포 구조를 건드려야 하고, 그 순간 "노뮤트 이식" 지시가 "예울마루 개편"으로 변질된다.
- **C안 탈락(현시점)**: 방향 자체는 원본 README의 이상론(소스 기반 워크플로)과 일치하나, ① Pages 빌드 커맨드 신설 = 기존 전 표면의 배포 경로에 신규 실패 지점 주입 ② 빌드 환경 민감성(README가 명시 경고)이 배포마다 발동 ③ 7분 컷 불가. **단, C안의 정신(산출물 재생성 가능성)은 A안 위에서 P1-3(러너 검증)으로 이미 흡수**했다 — A안은 C안으로 가는 길을 막지 않는다(소스가 전부 함께 이식됐으므로 언제든 전환 가능).
- **D안 탈락**: 최다 비용 + 규범 정면 위반(독립 툴톤 재색칠 영구 금지 · 임의 창작 금지). 논외.
- **A안 채택 근거의 뼈대**: ① 원본 설계 의도가 통폴더 이동("통째로 옮겨도 그대로 빌드" — 원작자 명시) ② 수용처 관례가 독립 표면 병렬 구조(스튜디오 문법)라 꽂기만 하면 됨 ③ 변형 0 = 검증 부담 0(게이트가 기계적으로 증명) ④ 이후 어떤 후속안(C 전환·계정 축·카드 연동)으로도 확장 가능한 **무손실 출발점**.
- **효율의 정의를 명시**하면: 효율 = (지시 충족도 × 미래 확장성) / (소요 시간 × 신규 리스크). A안은 분자 손실 없이 분모를 최소화한 유일안이다. 검증단은 이 논증(특히 "C안을 지금 안 한 것"과 "B안 전제 미성립 판정")을 우선 공격하라.
