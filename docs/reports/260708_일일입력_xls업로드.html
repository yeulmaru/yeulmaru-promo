#문제실측 근거
①셀러(티켓링크) 다운로드가 구형 .xls(OLE2/BIFF 이진)인데 업로드 코드가 .xlsx(JSZip=zip)만 처리 → 파싱 진입조차 안 됨매직바이트 D0CF11E0, file=CDFV2 Microsoft Excel
②열려도 서식 컬럼이 딴판 — 코드는 A='상품명:'/날짜·F~I 가정, 실제는 D=상품명·B=일자·현금 G/H·카드 I/Jschedule 시트 실측(조재혁 리사이틀 7/8 37석)
③공연 자동매칭 실패 시 저장 봉쇄 — 사용자가 직접 고를 방법 없음스크린샷 "이름 비슷한 게 여럿 → 저장 제외"

수정 전 / 후

변경 전

  • .xlsx만 허용 → .xls는 "xlsx만 올릴 수 있어요"
  • JSZip으로 sheet1.xml 직접 파싱(단일 서식 가정)
  • 자동매칭 실패 = 저장 제외(막다른 길)

변경 후

  • .xls·.xlsx 모두 — SheetJS 지연 로드(_ensureXLSX)
  • 티켓링크 「거래 일자별 집계」 파서 + 레거시 서식 자동감지, 상품명별 그룹
  • 카드마다 공연 선택 드롭다운 — 자동인식되면 그대로, 못 잡으면 직접 선택 시 저장. 미선택이면 "선택해야 저장·분석돼요" 안내

SheetJS는 이 앱이 이미 쓰는 cdn.jsdelivr.net(preconnect·폰트·MSAL 폴백·JSZip)에서 지연 로드 — 신규 의존 아님. 새 raw hex 0(토큰 계승, 게이트 1641/1641).
