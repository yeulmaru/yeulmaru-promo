# 260610 모달 감사 — B안 수정 내역

> ⚠️ 이전 세션이 저장했다는 `docs/260610_모달감사_버그목록.md`는 **임시 컨테이너에만 저장되고 푸시되지 않아 유실**됨 (git 전체 이력·OneDrive·Google Drive 검색 모두 없음).
> 본 세션에서 B안 범위(Critical 2건 + Major 핵심: 카카오 우회·닫기버튼·prefill·일괄처리 3종·copyRec)를 **코드 재감사로 재확정**한 뒤 수정함. 아래가 확정·수정된 버그 목록.

## Critical

1. **드래그 이동 = POST+DELETE → 중복 레코드 + 데이터 소실** (`onCellDrop`)
   - 새 행 POST 후 구 행 DELETE — DELETE 실패 시 중복 레코드 잔존.
   - 새 행이 18컬럼뿐이라 **신청자(S)·결과(T~V)·상태추적(W~X)·사유(Y~AA) 전부 소실**.
   - → 기존 행 **in-place PATCH** (`buildRecordRow`)로 전환.

2. **닫기(✕) 후 confirm 취소 → 수정이 신규 등록으로 둔갑(중복 레코드)** (`closePromoWizard`)
   - edit 상태(`_pwMode`/`_pwEditTarget`)를 confirm **전에** 리셋 → "닫을까요?"에서 아니오 선택 시 위저드는 열린 채 mode만 풀려, 저장 시 `submitPromoRequest`(POST)로 새 레코드 생성.
   - → 실제로 닫을 때만 상태 리셋.

## Major

3. **카카오 2일 연속 제한 우회** (`_kakaoSmsConflict`) — 당일·전날만 검사, **다음날 미검사** → 기존 카카오 신청 "하루 전"으로 신청하면 통과. → ±1일 모두 검사.
4. **변경 위저드 prefill 누락 → 콘텐츠 내용 소실** (`_prefillPwDataFromRec`) — kakaoText/folders/smsBody/타겟 등 본문 필드 미프리필 → 플랫폼 step 경유 시 `buildPwContentText()`가 빈 값으로 콘텐츠 내용 재생성. → 역파서 `_parseRecContent` 추가(구조화 텍스트 → 필드 복원, 문자 헤더/푸터 분리 포함).
5. **상태만 승인 시 카카오/문자 재검증 누락** (`submitPromoEdit`) — 일자·플랫폼 안 바꾸고 진행상태만 예정 전환(자동승인 포함)하면 충돌 검사 건너뜀. → 승인 전환 시 재검증 추가.
6. **일괄 접수**: 상태 가드 없음(신청 중 아닌 건도 예정 처리) + `canApprove` 미검증(카카오/문자 충돌 일괄 우회) + 신청자 알림 누락 → 3종 모두 수정.
7. **일괄 취소**: 종결(완료/취소) 건 덮어쓰기 + 신청자 알림 누락 → 스킵 가드 + 사유 포함 알림 추가.
8. **copyRec 신청자(S) 누락** — 복사본 18컬럼 POST → 신청자 비어 내신청 필터·알림·취소권한 단절. → 27컬럼(원본 신청자 유지) POST.
9. **saveEntry(레거시 모달)**: 신규 등록 신청자(S) 누락 + 수정 시 직전상태/상태변경KST 미추적 → 신규는 전체 컬럼, 수정은 `buildRecordRow`로 전환.

## 알림(메시지함) 전달 누락 보완 — 관리자 조치 → 신청자 알림

| 경로 | 기존 | 수정 |
|---|---|---|
| 단건 접수/보류/완료/반려/사유취소 | ✅ 있음 | - |
| **일괄 접수 / 일괄 취소** | ❌ 없음 | ✅ 추가 |
| **완료+결과입력**(`adminCompleteWithResult`) | ❌ 없음 | ✅ 추가 |
| **변경모달 [신청 취소]**(`cancelPromoRequest`, admin이 타인 건) | ❌ 없음 | ✅ 추가 |
| **변경 위저드 저장**(`submitPromoEdit`, admin이 타인 건 변경·상태변경) | ❌ 없음 | ✅ 추가 |
| **레거시 조회모달 승인**(`approvePromo`) | ❌ 없음 | ✅ 추가 |

배지/메시지함 폴링은 10초 주기(`_initBadge` setInterval) + 30초 API 스로틀로 정상 — 누락 원인은 전부 `pushMessage` 호출 부재였음. 수신자는 전 경로 **신청자(S)** 기준 유지.

## 미수정 (D안 후보 — minor)

- `records.length+1` No 채번 충돌(삭제 후 중복 No 가능) — 표시용이라 무해.
- 일괄 체크박스 `_pendingChecks`가 리로드 후 stale rowIndex 보유 가능(상태 가드로 오발사는 차단됨).
- 복사 시 진행 상태 원본 유지(보류 복사 시 보류사유 없는 보류 생성) — 제품 결정 필요.
- 기타 minor ~40건은 원목록 유실로 재감사 필요.
