## 🚨 제1 절대명령 — 디자인은 「루트 디자인 토큰」 계승 or 갱신, 둘 뿐 (레포 최상위 불변, 260704)

> ⚠️⚠️ 이 레포의 모든 디자인 변경(색·크기·간격·반지름·blur·모션·그림자·타이포·z·버튼·컴포넌트)은 예외 없이 아래 두 갈래 중 하나여야 한다. **세 번째 길은 없다.**
> - **① 계승(inherit) = 디폴트** — 필요한 값이 루트 디자인 토큰(**값 SSOT = `index.html`의 `:root` 2블록**)에 이미 있으면 그 토큰을 `var()`로 참조. **raw 색값 창작 절대 금지**(단, 같은 토큰의 alpha 변주는 정상 — 기틀 §3.5②). 구조값(radius·z·셀 등)도 **토큰이 있는 축이면 토큰 강제**(토큰 축이 없는 일반 px 레이아웃값은 기존 패턴 계승).
> - **② 갱신(update)** — `:root`에 없거나 기존 값을 바꿔야 하면 루트 토큰을 갱신. ⚠️ 갱신은 반드시 운영자에게 **먼저 묻고 승인**받은 뒤에만. **갱신의 특수형 = 운영자 승인 리터럴**: `var()`가 물리적으로 불가능한 위치(HTML `theme-color` 메타, baked PNG 등 빌드 에셋 색)는 승인 후 리터럴로 넣고 baseline/기틀에 사유 등재(§5 이력 전례) — 승인 없인 이 경로도 위반.
>
> **판정 절차(강제)**: 1) 필요한 값이 `:root`에 있나? → 있으면 계승(`var()`). 끝. 2) 없으면 → **멈추고 운영자에 묻는다**: 『기틀에 없다 — 갱신할지, 무엇을 계승할지(가장 가까운 기존 토큰 후보 제시)』. 임의 raw 박기 = 제1명령 위반. 3) 갱신 승인 → 즉시 편입 3점 세트: `:root` 토큰 추가/수정 → `docs/디자인기틀.md` 등재 → `tools/check_design.py` baseline 갱신+**사유 주석**.
>
> **색은 팔레트 폐쇄형** — `docs/디자인기틀.md` **§0 브랜드 컬러 확립본**의 컬러칩이 전부다(코어 인디고 `--accent` · 서브코어 살몬 `--c2` · 카테고리 6색 `--c1~--c6` · 역할색 각 딱 하나 · 중립 사다리). 칩 밖 색 반입·유사색 즉석 창작 = 위반. 시각본: `docs/브랜드컬러_확립본.html`.
>
> **왜**: raw 산개 = 하나 고칠 때 나머지 안 따라옴 = 품질 복리 붕괴 + 운영자 스트레스. 단일 토큰 SSOT + 계승/승인갱신만이 일관성 보장. **강제(기계)**: `tools/check_design.py`(baseline 래칫) + 편집 직후 훅(`design_gate.py`) + 커밋 게이트(`.githooks/pre-commit`) + 세션 계약 주입(`design_digest.py`).

### ✅ git (풀패스 + Start-Process + 파일 리다이렉트)
